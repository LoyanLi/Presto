# Presto 0.3.12

`0.3.12` 覆盖 `v0.3.11` 之后的全部变更。这个版本重点不是扩 DAW 能力面，而是把工具插件、启动状态、任务取消、配置持久化、更新显示和本地进程超时这些会直接影响日常使用稳定性的边界补齐。

## 重点变化

- 应用、workspace package、Tauri、Rust crate、contracts、SDK runtime 与 backend 版本基线统一到 `0.3.12`。
- 新增个人本地 `music-mix-project-tool` 工具插件，提供混音项目初始化入口、工程创建脚本、项目笔记模板和 revision log 模板；该插件位于 `plugins/local/`，不是官方插件。
- 个人本地 Music Mix Project Tool 页面完成一轮 UI 刷新，补齐工具页交互、执行状态、构建产物和工具核心测试。
- 宿主启动时的 DAW 状态不再在首次轮询完成前闪成 unavailable；设置页和侧边栏会保持更准确的 disconnected/unknown 语义。
- Jobs 取消链路统一：HTTP Jobs API、`jobs.cancel` capability、后台 job handle 与 DAW export cancellation 不再各走各的路径。
- Workflow 取消时会继续向正在 await 的 child job 传播取消请求，避免父任务显示 cancelled、子任务或 Pro Tools 操作仍在后台继续执行。
- Jobs 列表 `total_count` 改为筛选后的总数，不再被 `limit` 截断，便于后续分页和统计展示。
- `config.update` 改为先写 keychain、成功后再保存 config，避免 API key 保存失败时 UI 配置已经落盘。
- 更新页面和更新弹窗统一版本展示格式：GitHub release tag 的 `v0.3.x` 展示为 `0.3.x`，与应用当前版本一致。
- 更新检查请求增加连接超时和总时长超时，避免启动静默检查或手动检查在网络卡住时无限等待。
- 工具插件 bundled process 增加宿主侧超时边界；超时后宿主会终止子进程并返回 `BUNDLED_PROCESS_TIMEOUT`，避免 `tool.run` 永久挂起。
- Runtime capability list bridge 保留完整 capability metadata，Developer Console、Host 和 SDK runtime 不再收到被截断的 capability definition。
- `backend.daw-target.set` 由 Rust runtime 直接持久化 DAW target 并重启 backend，不再依赖正在被管理的 backend 完成配置写入。
- Rust runtime 领域状态继续下沉到 backend、plugins、mobile progress 模块，根文件只保留装配、helper 和 dispatch。
- 默认 app config 改为从 manifest 事实源生成 TypeScript、Python、Rust 三端产物，避免默认配置结构多处手写漂移。
- workspace package 入口统一到 `package.json#exports`，移除会造成包根解析语义不一致的重复转发文件。

## 个人本地 Music Mix Project Tool

- 新增个人本地插件目录 `plugins/local/music-mix-project-tool`；这是 `loyan.music-mix-project-tool`，不是 `plugins/official/` 下的官方插件。
- 插件 manifest 声明为 `tool` 类型，不绑定 DAW，面向本地文件和 shell runner 场景。
- 新增 `resources/scripts/create_project.sh`，作为当前工具执行项目目录初始化的脚本入口。
- 新增 `resources/templates/project-notes.md` 和 `resources/templates/revision-log.md`，为项目笔记和修改记录提供默认模板。
- 新增 `toolCore` 与 plugin module 测试，覆盖路径解析、执行参数和插件导出契约。
- 插件 dist 产物已进入仓库，宿主可以从本地插件目录直接加载该工具页面。

## 稳定性修复

### 启动状态

- `useDawStatusPolling` 和 Tauri host shell 启动装配修复了首次打开时 DAW 状态短暂误报 unavailable 的问题。
- 设置页中的 DAW 状态测试覆盖 unknown startup state，确保启动期不会把未知状态渲染成不可用。

### Jobs 与 Workflow 取消

- 新增统一取消 helper，让 HTTP `/jobs/{job_id}/cancel` 和 `jobs.cancel` capability 共享同一行为。
- 取消导出 job 时会同时设置后台 thread handle 的 cancel event，并调用 DAW adapter 的 `cancel_export()`。
- workflow await child job 时，如果父 workflow 被取消，会对 child job 执行同一套 managed cancel，确保 child import/export worker 能看到取消事件。
- 相关测试覆盖 HTTP 取消导出、capability 取消导出、workflow 取消 awaited child job 三类路径。

### Jobs 统计

- `InMemoryJobManager.list()` 现在先计算筛选后的 `total_count`，再应用 `limit`。
- 返回列表可以被截断，但 `total_count` 保持完整筛选总数。

### Config 与 Keychain

- `config.update` 不再先保存 config 再写 keychain。
- 当 API key 写入 keychain 失败时，config 不会落盘，避免偏好显示已启用但实际没有可用密钥。

### 更新版本显示

- 新增 `formatVersionLabel()`，只用于 UI 展示层去掉版本号前缀 `v` / `V`。
- 不改变 GitHub release tag 原始数据，也不改变已有版本比较逻辑；比较逻辑继续支持带 `v` 的 tag。
- GitHub release 检查使用 `curl --connect-timeout 5 --max-time 15`，把网络无响应限制为普通更新检查失败，而不是让 Tauri invoke 无限等待。

### 插件工具进程

- `process.execBundled` 改为通过宿主统一的 timed capture 执行 bundled script/binary。
- bundled process 默认最多运行 120 秒；超时后宿主会 kill 子进程、收集已有 stdout/stderr，并返回 `BUNDLED_PROCESS_TIMEOUT`。
- 这个边界只影响插件工具进程，不改变 macOS Accessibility、backend HTTP 或更新检查的既有执行路径。

## 架构与协议边界

### Capability metadata

- Rust runtime 的 capability list 转换现在保留后端 schema 中的完整 metadata。
- `workflow_scope`、`portability`、`implementations`、`field_support` 等后端 snake_case 字段会映射成 SDK runtime 使用的 camelCase 字段。
- 这保证 Host、Developer Console 和插件开发工具看到的 capability definition 与 `BackendCapabilityDefinition` 类型契约一致，不再基于缺字段的局部能力视图做判断。

### DAW target 持久化

- `backend.daw-target.set` 不再通过 `config.get` / `config.update` capability 读写目标 DAW。
- Rust runtime 会直接更新应用数据目录下 `config.json` 的 `hostPreferences.dawTarget`。
- 持久化成功后，runtime 再执行 `stop -> set target -> start -> wait ready`，让 backend supervisor 的启动输入不反向依赖 backend 自己。

### Rust runtime 模块边界

- `src-tauri/src/runtime.rs` 继续收口为根级装配、共享 helper 和 operation dispatch。
- backend supervisor state 归入 `runtime/backend.rs`。
- 插件候选模型和 workflow definition 引用归入 `runtime/plugins.rs`。
- mobile progress session state 归入 `runtime/mobile_progress.rs`。
- 新增 runtime operation 时应优先落到对应领域模块，再由根 dispatch 暴露。

### 默认配置事实源

- 默认 app config 的手写事实源改为 `packages/contracts-manifest/app-config-defaults.json`。
- `scripts/generate-contracts.mjs` 会生成：
  - `packages/contracts/src/generated/appConfigDefaults.ts`
  - `backend/presto/application/app_config_defaults_generated.py`
  - `src-tauri/src/runtime/app_config_defaults_generated.rs`
- `hostPreferences`、`uiPreferences` 这类默认结构不再分别散落在 TypeScript 类型、Python config store 和 Rust runtime 读取逻辑里。

### Package exports

- `packages/contracts/index.ts`、`packages/sdk-runtime/index.ts` 和 `packages/sdk-runtime/createPrestoRuntime.ts` 这些包根重复转发文件已移除。
- workspace package 的公开入口以各自 `package.json#exports` 为准。
- 这样可以避免工具或构建器按文件路径解析包根时拿到另一套入口语义。
