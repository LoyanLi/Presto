# Presto 0.3.8

`0.3.8` 是当前正式发布版本。这一版继续收口官方 `tool` 页面和现有 workflow 壳层的设计语言，重点把 `official.atmos-video-mux-tool` 调整为更简洁、更可复用的两步工具流。

## Highlights

- `official.atmos-video-mux-tool` 页面继续向 workflow UI 基线收口：保留宿主标题，内部改为直接复用共享 stepper、panel、输入控件和底部 action bar。
- Atmos mux 工具当前正式改为两步流：`Sources` 与 `Output / Review / Run`，底部前进 / 后退 / 运行动作固定在窗口底部，滚动只发生在页面主体内容区。
- 输出与结果区域进一步精简：移除顶部 `Video / Atmos / Output` 状态条、重复路径块、`jobId` badge、冗余 `Job / Output` chips 与完整输出路径回显。
- 成功完成后的正式结果表达只保留紧凑状态和输出文件名，页面不再把本地完整路径作为主要 UI 内容回显。
- `tool` 插件技术文档同步补充当前正式 UI 约束：tool 页面优先复用 workflow shell，避免重复标题、重复路径和脱离宿主基线的底部工具栏。

## Upgrade Notes

- 根 `package.json` 仍然是唯一手工维护的产品版本源；workspace package、Tauri 配置、Rust crate、contracts 版本常量和 backend 版本常量继续通过同步脚本派生。
- 本次调整不改变 `official.atmos-video-mux-tool` 的 manifest、runner 权限边界和 bundled resources，只调整正式页面组织方式与结果展示密度。
- 如果后续新增 `tool` 页面，当前默认基线不是“单独设计一套工具页壳层”，而是先复用现有 workflow stepper / panel / action bar 语义，再补 tool 自身输入和运行逻辑。

## Artifacts

本轮提交只完成版本、页面与文档基线更新；当前 release note 先保持与前几版相同结构，但未在这一提交内重新生成新的正式安装包校验值。

- 下一次正式打包 `0.3.8` 安装包时，应补齐双架构 `dmg` 文件名、`SHA-256` 和 `Size bytes`。
- 如果沿用当前本地打包链，构建日志仍会出现未签名构建提示；这一点与 `0.3.7` 的本地产物策略一致。
