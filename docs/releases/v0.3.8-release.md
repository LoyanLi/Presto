# Presto 0.3.8

`0.3.8` 是当前正式发布版本。这一版把 `tool` 插件表面从 contracts、host、runtime 到官方样例一起接通，同时继续把 `official.atmos-video-mux-tool` 页面收口到现有 workflow 壳层。

## Highlights

- `tool` 插件当前正式成为一等插件类型：contracts、host runtime、Tauri runtime 与插件装配链已经接通 `tool.run`、页面 host 能力和 bundled process 执行链。
- 宿主新增 `Tools` 页面和 `Tool Extensions` 管理入口；tool 插件不再混进 workflow 首页，而是走独立工具表面与独立扩展管理。
- 新增官方 `official.atmos-video-mux-tool` 参考插件，完整覆盖 manifest、tool runner、`bundledResources`、`process.execBundled(...)` 与本地工具链打包闭环。
- `official.atmos-video-mux-tool` 页面继续向 workflow UI 基线收口：改为两步流 `Sources` / `Output / Review / Run`，复用共享 stepper、panel、输入控件和底部 action bar。
- 输出与结果区域进一步精简：移除顶部 `Video / Atmos / Output` 状态条、重复路径块、`jobId`、冗余结果 chips 与完整输出路径回显；成功后只保留紧凑状态和输出文件名。
- 宿主运行统计继续接入 `tools` 维度，成功 tool job 现在会进入 runs metrics。

## Upgrade Notes

- 根 `package.json` 仍然是唯一手工维护的产品版本源；workspace package、Tauri 配置、Rust crate、contracts 版本常量和 backend 版本常量继续通过同步脚本派生。
- `tool` 插件当前正式边界是：`extensionType: "tool"`、`supportedDaws: []`、`pages[].mount: "tools"`、`tools[]`、`toolRuntimePermissions`、`bundledResources`。
- 如果后续新增 `tool` 页面，当前默认基线不是再造一套独立工具页壳层，而是优先复用 workflow stepper / panel / action bar 语义，再补 tool 自身输入和运行逻辑。
- 本版本也把仓库许可证基线明确统一为 `AGPL-3.0-only`。
