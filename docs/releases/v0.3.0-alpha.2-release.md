# Presto 0.3.0-alpha.2

Presto `0.3.0-alpha.2` 是桌面宿主从 Electron 正式切换到 Tauri 后的下一轮收口版本。

这一版的核心不是继续加功能，而是把桌面运行主线、插件执行边界、workflow 宿主装配、主题能力和 macOS 打包链统一到同一条正式发布路径。

## 亮点

### Tauri 桌面主线正式建立

- 桌面宿主正式切换为 `Tauri + Node sidecar + Python FastAPI`
- `src-tauri/` 成为唯一桌面宿主入口
- `frontend/sidecar/` 成为唯一业务宿主进程
- Electron 正式运行链退出主线

### Workflow 与插件边界收口

- workflow 插件执行边界统一为 `frontend -> sidecar -> backend`
- 前端只提交声明式 workflow 输入
- sidecar 负责 workflow definition、allowed capabilities 和宿主装配
- backend 负责 capability 执行与 workflow 展开后的任务运行

### Host UI 与主题整理

- Host UI 的 workflow 设置流程完成统一
- 下拉、导入入口、文件夹选择与扫描链路统一
- 接入全局浅色 / 深色主题切换
- 启动 splash 和主窗口白屏感知进一步优化

### 打包链与包体瘦身

- 打包链固定为 `frontend build + sidecar build + runtime resources + tauri bundle`
- sidecar 内置 Node 改为按目标架构裁剪并去符号
- backend、插件和 automation 资源改为筛选后入包
- 现已支持独立的 `arm64` 与 `x64` macOS 安装包构建

## 修复

- 修复 Tauri 主线下 workflow definition / allowedCapabilities 装配缺失问题
- 修复 import / export workflow 输入命名与 payload 对齐问题
- 修复 workflow 页面若干下拉与列表样式不统一问题
- 修复打包态下后端、插件和 automation 资源路径定位
- 修复 macOS 应用图标资源链路
- 修复 Import Workflow 进度阶段与实际执行顺序不一致的问题，导入后阶段现按 `rename -> color -> strip` 批次运行
- 修复 workflow executor 顶层阶段进度汇报，导入后不再被子步骤 `rename_track` / `apply_color` / `strip_silence` 覆盖
- 修复官方插件播种状态兼容旧版版本号种子值的问题，更新后的官方插件定义可以正确刷新到受管扩展目录
- 修复 `tauri dev` 读取 `target/debug/resources` 旧资源快照的问题，开发态 sidecar 现直接读取仓库根目录资源

## 安装包

- `arm64` DMG：`release/tauri/arm64/Presto_0.3.0-alpha.2_arm64.dmg`，当前实测约 `66 MB`
- `x64` DMG：本次未重新构建，需在 `npm run tauri:build:x64` 后补充对应产物与大小

## 当前发布边界

- 当前仅支持 macOS
- 当前正式目标 DAW 为 `Pro Tools`
- mobile progress 仍保留 sidecar 内本地 HTTP 服务
- 包体剩余大头仍来自内置 Node sidecar，本版本未改变这一架构前提
