# Presto 0.3.0

Presto `0.3.0` 是桌面宿主从 Electron 完整切换到 Tauri 后的首个正式发布版本。

这一版的目标不是再扩张范围，而是把当前已经成立的桌面主线、workflow 与插件边界、资源打包链以及 macOS 发布形态收口为一套可以稳定复用的正式发布结果。

## Highlights

### Tauri 桌面主线正式落版

- 桌面宿主正式固定为 `Tauri + Node sidecar + Python FastAPI`
- `src-tauri/` 成为唯一桌面宿主入口
- `frontend/sidecar/` 成为唯一业务宿主进程
- Electron 运行链退出当前正式发布路径

### Workflow 与插件边界收口

- workflow 执行路径统一为 `frontend -> sidecar -> backend`
- 前端负责提交声明式输入
- sidecar 负责 workflow definition、allowed capabilities 和宿主装配
- backend 负责 capability 执行与展开后的任务运行

### Host UI 与主题能力整理

- workflow 设置页、导入入口、文件夹选择与扫描链路统一
- 接入全局浅色 / 深色主题切换
- splash 与主窗口启动阶段的白屏感知进一步收口

### macOS 发布图标链稳定化

- 当前 Tauri 发布包直接复用旧 Electron 成功产物里的 `Assets.car`
- 同步复用旧包的 `icon.icns`
- `Info.plist` 对齐到旧包已验证的 `CFBundleIconName=Icon` 与 `CFBundleIconFile=icon.icns`

## Fixes

- 修复 Tauri 主线下 workflow definition / allowed capabilities 装配缺失问题
- 修复 import / export workflow 输入命名与 payload 对齐问题
- 修复 workflow 页面若干下拉与列表样式不统一问题
- 修复打包态下后端、插件和 automation 资源路径定位
- 修复 Import Workflow 阶段进度与实际执行顺序不一致的问题
- 修复 workflow executor 顶层阶段进度被子步骤覆盖的问题
- 修复官方插件播种状态与旧版版本号种子兼容问题
- 修复 `tauri dev` 读取 `target/debug/resources` 旧快照的问题

## Upgrade Notes

- 当前仅支持 macOS
- 当前正式目标 DAW 为 `Pro Tools`
- mobile progress 仍保留 sidecar 内本地 HTTP 服务
- 包体剩余大头仍来自内置 Node sidecar，本版本未改变这一架构前提
