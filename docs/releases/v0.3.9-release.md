# Presto 0.3.9

`0.3.9` 是当前正式发布版本。这一版继续收口宿主里的 `Runs` 统计交互，同时补入一个纯前端官方工具插件样例。

## Highlights

- `Runs` 页面交互从“四份榜单同时展开”改为“默认总览 -> 单类详情”：先显示 `workflow`、`automation`、`tool`、`command` 四类总览卡片，再进入单类详情榜单。
- `Runs` 总览卡片当前直接显示累计次数、最高频项和最近使用时间，页面主次更明确，不再依赖并列四卡滚动去传达统计结果。
- 新增官方 `official.time-calculator-tool`，在 `Tools` 页提供 `BPM -> Time`、`Time -> BPM`、`Reverb / Pre-delay` 三组计算。
- `official.time-calculator-tool` 是当前纯前端 tool 插件的正式参考：不声明 capability、不依赖 runner，插件内部自带本地化与样式，长列表限制在卡片内滚动。

## Upgrade Notes

- 根 `package.json` 仍然是唯一手工维护的产品版本源；workspace package、Tauri 配置、Rust crate、contracts 版本常量和 backend 版本常量继续通过 `scripts/sync-version.mjs` 派生。
- `Runs` 页面当前不是同时保留四类完整榜单，而是保留一个总览层和一个单榜单详情层；如果后续继续加统计维度，应先进入总览卡片和详情切换模型，而不是恢复并列长列表。
- `tool` 插件当前除了 bundled process 工具型样例，也已经有纯前端计算型样例；是否需要 runner，应以工具是否真的要调用外部执行链为边界。
