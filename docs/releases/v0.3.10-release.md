# Presto 0.3.10

`0.3.10` 是当前正式发布版本。这一版重点不是扩新功能面，而是把导出执行链路、运行时日志和宿主调用边界收紧到更可诊断、更不容易把桌面宿主拖死的状态。

## Highlights

- 原始 runtime log 现在直接接入 execution lifecycle 事件；后端、Tauri runtime 和插件宿主会把执行事件写进同一条诊断链，而不是再分散到彼此看不到的输出面。
- execution log 落盘格式收紧为单行可读摘要，必要上下文以内联紧凑字段追加；导出链路日志终于能直接扫读和 `grep`，而不是再被多行 JSON 撕碎。
- 导出 workflow 补齐了明确的生命周期事件：`export.run.accepted`、`export.run.started`、`export.file.succeeded`、`export.run.succeeded`、`export.run.failed`，导出过程中终于能看到“接受任务 -> 开始 -> 单文件完成 -> 整体结束”的真实顺序。
- 低信号成功态日志被收紧：`daw.connection.getStatus`、`daw.adapter.getSnapshot`、`daw.session.getInfo`、`daw.track.list`、`daw.export.mixWithSource` 和 `jobs.get` 不再把 runtime log 刷成轮询噪音，失败态仍然保留。
- 插件宿主写 runtime log 时会抑制紧邻的重复激活日志；官方插件 activation 在原始日志里不再成对重复出现。
- backend supervisor 现在会正确识别 backend `stderr` 的级别，并清理 ANSI 控制字符；`Uvicorn INFO` 不会再被误记成 `error`，`lastError` 也不会再被启动信息污染。
- Tauri `runtime_invoke` 已改成异步命令，并把 `backend.*` 调用移到 `spawn_blocking`；导出期间慢 invoke 不再直接占住宿主命令处理线程，这是这轮针对导出彩虹圈的核心收口。

## Upgrade Notes

- 根 `package.json` 仍然是唯一手工维护的产品版本源；workspace package、Tauri 配置、Rust crate、contracts 版本常量和 backend 版本常量继续通过 `scripts/sync-version.mjs` 派生。
- 当前日志设计仍然坚持“写回原始 runtime log”，这一版没有新增单独日志中心或第二套运行日志 UI。
- 这一版的导出问题修正重点是“提升真实状态可观测性”和“避免宿主 IPC 因阻塞 backend invoke 假死”；它不是重新设计导出业务流。
- `jobs.get` 依旧保留为公开 job 查询能力，只是成功轮询不再写 execution lifecycle；如果需要排查 job 异常，仍然看失败态和导出生命周期事件。
