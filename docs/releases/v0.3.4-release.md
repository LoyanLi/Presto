# Presto 0.3.4

`0.3.4` 是当前正式发布版本。这一版把正式安装包资源同步修复、Rust runtime 后台启动稳定性修复、单窗口启动链路收口，以及本轮文档与发布材料整理一起收口到正式版。

## Highlights

- 修复 Tauri 正式打包后 `.app` bundle 漏掉 `backend`、`frontend`、`plugins` 资源的问题，避免用户拿到只有壳层、缺少运行时资源的错误安装包。
- 修复 Rust runtime 本地 HTTP 健康检查错误关闭 socket 写端的问题，避免后端已经启动成功后又被宿主误判失败并主动清理，导致主页面之外的功能全部不可用。
- Tauri 启动链改为单窗口主界面启动，移除独立 splashscreen 窗口；主窗口直接承载启动壳层，启动路径更直接，也更稳定。
- 继续沿用自包含 bundled Python runtime，保持不依赖用户机器本地 `/Library/Frameworks/Python.framework`，并保留上一轮 Python runtime 裁剪结果。
- `0.3.4` 双架构正式安装包已经重新完成构建和验证：`arm64` 与 `x64` 都能生成完整 `.app` / `.dmg` 产物，并恢复到真实可安装、可启动状态。
- macOS 辅助功能权限仍会在每次启动时主动预检；缺权限时会先弹出明确引导，而不是只返回底层脚本错误。

## Upgrade Notes

- 当前更新机制仍然是“检测新版本并引导跳转下载”，不是增量补丁更新。
- 当前实际支持的 DAW 目标仍然只有 `pro_tools`。
- 首次使用自动化前，macOS 仍然需要在“系统设置 -> 隐私与安全性 -> 辅助功能”中为 `Presto` 授权。
- 当前正式发布主线文档已经统一更新到真实架构：`Tauri + Rust runtime + React + Python FastAPI`。
