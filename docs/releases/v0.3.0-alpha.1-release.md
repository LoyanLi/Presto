# Presto 0.3.0-alpha.1 Release Notes

发布日期：2026-04-03

## 概述

`0.3.0-alpha.1` 是 Presto 桌面宿主从 Electron 正式切到 Tauri 的收口版本。

当前正式拓扑固定为：

- `src-tauri/` 作为唯一桌面宿主入口
- `frontend/sidecar/` 作为唯一 Node sidecar 业务宿主
- `backend/presto/` 继续作为 Python FastAPI 本地后端

这个版本的目标不是继续叠加功能，而是把插件边界、workflow 调度、桌面宿主、Host UI、主题能力和 macOS 打包链统一到同一条正式发布路径。

## 本次发布内容

### 1. 插件与 workflow 执行边界收口

- workflow 插件改为声明式执行边界，前端不再直接拼装执行细节
- sidecar 负责 workflow definition、allowed capabilities 和宿主装配
- backend 负责 capability 执行和 workflow 展开后的任务运行
- 插件 contract、manifest、capability 请求结构同步整理

对应提交：

- `refactor(plugin-boundary): move execution to backend`

### 2. Tauri 宿主主线建立

- Electron 正式运行链退出主线
- Tauri Rust 壳负责窗口、应用元数据、系统 API 和 sidecar 生命周期
- Node sidecar 负责 backend supervisor、plugin host、automation、mac accessibility、mobile progress 和日志链路
- 前端桌面入口统一到宿主无关渲染层，不再依赖旧 preload 运行模型

对应提交：

- `feat(tauri-host): migrate desktop runtime`

### 3. Host UI、workflow 交互与主题整理

- workflow 页面内的下拉、列表、导入入口和文件夹选择交互统一
- import workflow 的 source folder 浏览和自动扫描链路整理
- 全局浅色 / 深色主题切换接入桌面宿主
- splash 和启动体验修正，减少主窗口启动过程中的白屏感知

对应提交：

- `feat(host-ui): unify workflow setup and theme`

### 4. 发布文档与开发文档重组

- 文档入口改为当前代码结构对应的 `docs/kernel-development/` 和 `docs/plugin-development/`
- README、架构文档、插件开发文档和 release note 统一为当前主线事实
- 移除旧 Electron 时代的平铺式架构页

对应提交：

- `chore(release): v0.3.0-alpha.1`

### 5. Tauri 包体瘦身

- sidecar Node 二进制改为按当前架构裁剪并去符号
- backend、官方插件和 automation 资源改为预筛选后再入包
- 应用图标改为新的 macOS 打包资源

实测结果：

- `Presto.app`：约 `96M`
- `Presto_0.3.0-alpha.1_aarch64.dmg`：约 `36M`

对应提交：

- `chore(tauri-bundle): slim app package`

## 发布形态

- 平台：macOS
- 宿主：Tauri + Node sidecar + Python FastAPI
- 发布物：`.app` 与 `.dmg`
- 版本：`0.3.0-alpha.1`

## 适用对象

- 需要验证 Tauri 桌面主线是否可作为正式宿主运行的内部开发者
- 需要基于当前插件 ABI、manifest 和 capability contract 开发插件的开发者
- 需要验证 macOS 安装包、主题、workflow、automation 和宿主链路一致性的测试人员

## 已知边界

- 当前目标平台仅 macOS
- mobile progress 仍保留 sidecar 内本地 HTTP 服务
- 包体的主要剩余体积来自内置 Node sidecar，本版本没有改变这一架构前提
