# Presto 0.3.0-alpha.1 发布文案

发布日期：2026-03-30

## 版本概述

`0.3.0-alpha.1` 是一次结构性版本更新。这个版本把 Presto 从“围绕单一导入流程扩展的桌面工具”进一步收敛为一套清晰的三层运行结构：

- Electron 桌面宿主
- 本地 FastAPI 后端能力层
- 受权限约束的插件扩展层

这次版本的重点不是增加花哨入口，而是把运行时边界、共享契约、插件接入方式和发布打包路径统一到一个更稳定的形态上，为后续继续演进工作流和插件生态打基础。

## 本次重点

### 1. 运行时边界重整

当前能力调用统一通过宿主桥接进入本地后端 `/api/v1` 能力入口，宿主、后端与插件不再各自维护一套松散边界。目录结构、文档和运行模型现在表达的是同一套事实。

### 2. 契约层与插件系统成型

仓库已经形成以下正式分层：

- `packages/contracts`
- `packages/contracts-manifest`
- `packages/sdk-core`
- `packages/sdk-runtime`
- `host-plugin-runtime`
- `plugins/official/*`

这意味着 Presto 当前的插件接入不再是“任意脚本直连宿主”，而是 manifest、权限白名单、capability 契约与 runtime service 白名单共同驱动的受控模型。

### 3. 后端目录语义收敛

后端主目录已经从 `backend/import/presto` 收敛为 `backend/presto`。原因很直接：这里承载的是整个 Presto 本地后端，而不是某个“import 子系统”。这次调整消除了目录命名和真实职责之间的偏差。

### 4. macOS 打包链路整理

当前发布产物已经拆分为双架构安装包：

- `Presto-0.3.0-alpha.1-arm64.dmg`
- `Presto-0.3.0-alpha.1-x64.dmg`

同时，打包输入改为更小的最小文件集，并继续使用 `asar` 与最大压缩。后端资源通过 `extraResources/backend` 进入安装包，安装态路径解析已同步到当前目录结构。

## 修复与改进

- 修复打包态下后端资源根路径解析，确保应用能从安装包内正确定位 `backend/presto`
- 修复 macOS 图标打包链路，规避 `.icon` 输入触发的 `actool` 资产编译失败
- 修复启动后应用图标回跳问题，移除 Dock 图标覆写路径
- About 面板与应用版本展示现在统一读取 Presto 自身版本信息，而不是 Electron 运行时版本

## 当前已知发布形态

- 平台：macOS
- 目标 DAW：当前实际稳定目标仍为 `pro_tools`
- 发布类型：未签名 DMG
- 安装包：
  - `arm64` 约 100 MB
  - `x64` 约 103 MB

## 适合谁升级

这个版本更适合以下几类人：

- 需要验证新的宿主 / 后端 / 插件边界是否稳定的内部开发者
- 需要基于正式 capability / runtime service 契约接入插件的开发者
- 需要分别在 Apple Silicon 与 Intel macOS 机器上验证安装包的测试人员

如果你的关注点是“当前 Presto 的系统形态是否已经收敛出一条清晰主线”，`0.3.0-alpha.1` 就是这个拐点版本。
