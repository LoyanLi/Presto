# Presto 0.3.0-alpha.1 发布文案

发布日期：2026-04-02

## 版本概述

`0.3.0-alpha.1` 是当前 Tauri 主线的收口版本。这个版本把桌面宿主正式固定为：

- Tauri Rust 宿主
- Node sidecar 业务宿主
- Python FastAPI 本地后端

这次版本的重点不是继续堆功能，而是把运行时边界、插件协议、workflow 宿主装配、主题切换和 macOS 打包链路统一到同一条正式发布路径上。

## 本次重点

### 1. 桌面宿主切换正式收口

当前桌面运行主线已经收口到：

- `src-tauri/`
- `frontend/tauri/`
- `frontend/sidecar/`

Tauri 负责窗口和系统能力，Node sidecar 负责业务宿主，Python backend 继续承接 capability 与 job 执行。

### 2. 插件 runtime 与 workflow 装配收口

当前 workflow 插件的正式调用路径已经统一为：

- 前端只发 `pluginId + workflowId + input`
- sidecar 负责解析受信任 workflow definition 与 allowed capabilities
- backend 只执行已展开的 declarative workflow

这次同时修正了 import/export workflow 在 Tauri 主线下暴露出来的宿主装配与输入命名问题。

### 3. 文档结构重组

文档不再保留旧的平铺架构页，而是收口为两块主入口：

- `docs/kernel-development/`
- `docs/plugin-development/`

README 与插件协议文档现在都只描述当前代码已经成立的事实。

### 4. 主题与启动体验整理

当前版本已经包含：

- 全局浅色 / 深色主题切换
- 启动 splash 流程整理
- Tauri 主窗口冷启动白屏优化
- 深色模式下右上角 Presto 图标反相

## 修复与改进

- 修复 import workflow 在结构化宿主错误下显示 `[object Object]`
- 修复 workflow.run.start 在 Tauri sidecar 下缺失 definition / allowedCapabilities 的宿主装配问题
- 修复 export workflow payload 命名不一致导致的 workflow 引用失败
- 统一文档对插件 host、runtime、capability 边界的描述
- 统一应用版本来源到当前 Tauri 配置与包版本

## 当前发布形态

- 平台：macOS
- 目标 DAW：`pro_tools`
- 发布类型：Tauri macOS DMG
- 版本：`0.3.0-alpha.1`

## 适合谁升级

- 需要验证 Tauri 正式宿主链是否已经可作为主线运行的内部开发者
- 需要基于当前插件协议开发 workflow / automation 插件的开发者
- 需要验证 macOS 安装包与主题、workflow、宿主边界是否一致的测试人员
