# Presto 0.3.7

`0.3.7` 是当前正式发布版本。这一版把宿主里的运行统计单独做成 `Runs` 页面，同时补上产品落地页和本地静态预览入口。

## Highlights

- Host shell 新增 `Runs` 页面，按 `workflow`、`automation`、`command` 三个维度展示成功运行排行。
- workflow 统计口径改为“成功完成的 workflow job”；`workflow.run.start` 不再计入 workflow 榜，但仍然计入 command 榜。
- 成功 workflow job 现在会把内部成功执行的 command 次数一起沉淀到运行统计里，命令榜能反映 workflow 内部实际调用频率。
- command 榜支持宿主侧译名显示，不再直接暴露原始 capability ID。
- `Runs` 页面布局改成和现有 host shell 一致的卡片语言，并把滚动限制在卡片内部；宽度足够时固定三列，否则退回单列。
- 仓库新增 `presto-product-landing/` 静态产品落地页，并提供 `npm run landing:preview` 作为本地预览入口。

## Upgrade Notes

- 根 `package.json` 仍然是唯一手工维护的产品版本源；workspace package、Tauri 配置、Rust crate、contracts 版本常量和 backend 版本常量继续通过同步脚本派生。
- 如果要本地查看营销落地页，不需要走桌面宿主构建，直接运行 `npm run landing:preview` 即可。
- `Runs` 页面当前只统计成功执行，不把失败执行写进排行结果。
