# Presto 0.3.6

`0.3.6` 是当前正式发布版本。这一版继续保持 `0.3.x` 只正式支持 `Pro Tools` 的边界，同时把 capability registry、PTSL 语义封装和插件声明面统一到同一套公共语义。

## Highlights

- public capability ID 统一收口到 `daw.*` 命名空间，workflow / automation manifest 与 declarative workflow `usesCapability` 不再混用旧命名和 DAW 特化命名。
- 生成产物、后端 handler registry、SDK client、插件 manifest、Host Developer Console 与 permission guard 现在都围绕同一份 canonical capability registry 工作。
- `PTSL` 继续只作为 backend-private 的 `Pro Tools` 内部命令层扩面；public capability 面保持语义封装，不把每个 `PTSL` 命令直接暴露成一对一公开 capability。
- 现有 workflow / automation 的运行时调用接口保持不变，继续复用 `presto.session.*`、`presto.track.*`、`presto.import.*`、`presto.export.*`、`presto.automation.*` 等稳定入口。
- 宿主 DAW 状态轮询修复为 canonical `developerPresto.session.getInfo()` 调用，不再访问不存在的 `developerPresto.daw.session.getInfo()`。
- `official.import-workflow` 现在支持显式选择导入音频模式：`copy` 或 `link`。
- `official.import-workflow` 现在支持在 `Strip Silence` 后批量创建 fades；当 `Fade preset name` 为空时，后端会省略 `fade_preset_name`，让 `Pro Tools` 直接使用 Fades 对话框中的 last-used 设置。

## Upgrade Notes

- `0.3.x` 当前仍然只正式支持 `pro_tools`；`logic`、`cubase`、`nuendo` 仍然只是预留目标，不在这一版接通运行时。
- 如果在 `0.3.x` 里谈“完整覆盖 PTSL”，它只应该指 `Pro Tools` 内部命令层覆盖，不意味着 public capability surface 与 `PTSL` 命令名一一对应。
- 仓库根 `package.json` 仍然是唯一手工维护的产品版本源；workspace package、Tauri 配置、Rust crate 和 backend 版本常量都由同步脚本派生。
- 对 import workflow 而言，空字符串不再被当成非法 fade preset 名称；它现在有明确语义，表示“使用 Pro Tools 当前 last-used fade settings”，而不是“查找一个名为空的 preset”。
