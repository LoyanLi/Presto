# Presto 0.3.3

`0.3.3` 是当前正式发布版本。这一版把原先准备放进 `0.3.3-alpha.1` 的更新检查、打包态 Python runtime 便携化修复、安装包体积优化，以及本轮 macOS 辅助功能权限引导一起收口到正式版。

## Highlights

- 新增应用启动自动检查 GitHub Releases，并支持用户选择是否包含预览版。
- 宿主发现新版本后会弹出更新提示，用户可以直接跳转到发布页下载。
- 修复打包态 bundled Python runtime 仍外链用户机器 `/Library/Frameworks/Python.framework` 的问题，避免新机器上因缺少本地 Python framework 导致后端启动失败。
- 修复 `x64` 安装包错误复用 `arm64` Python wheels 的问题，避免 Intel Mac 或 Rosetta 下因 `_pydantic_core` 架构不匹配导致后端启动失败。
- bundled Python runtime 进一步收口为随包自包含形态，并剔除测试、开发和桌面 GUI 冗余内容，显著缩小 `.app` 与 `.dmg` 体积。
- macOS 辅助功能权限现在会在每次启动时主动预检；任何依赖 Accessibility 的运行时调用在权限缺失时都会先弹出明确引导，而不是只返回底层 `osascript` 报错。

## Upgrade Notes

- `0.3.3-alpha.1` 没有单独发布，正式用户直接使用 `0.3.3` 即可。
- 当前更新机制仍然是“检测新版本并引导跳转下载”，不是增量补丁更新。
- 当前实际支持的 DAW 目标仍然只有 `pro_tools`。
- 首次使用自动化前，macOS 仍然需要在“系统设置 -> 隐私与安全性 -> 辅助功能”中为 `Presto` 授权。
