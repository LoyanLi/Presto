# Presto 0.3.1

Presto `0.3.1` 是 `0.3.0` 之后的一次功能与修复更新。

这一版不改桌面主线架构，重点是把插件宿主里的 extension management、automation 宿主卡片、轨道状态操作封装和官方批量备份 automation 插件收口到同一套当前可继续迭代的事实基础上。

## 亮点

### Extension management 从硬编码入口切回安装插件事实

- 宿主设置页现在分别展示 `Workflow Extensions` 与 `Automation Extensions`
- workflow / automation 扩展统一接到同一套 plugin host 管理链
- 管理动作现在统一覆盖刷新、安装目录、安装 zip、启用、禁用与已安装扩展卸载
- automation 宿主页面不再把 `splitStereoToMono` 当作唯一固定入口，而是按当前已安装 automation 插件渲染

### Automation 宿主与官方批量备份插件落版

- automation 首页卡片改为按内容各自独立高度布局，不再被最长卡片统一拉伸
- 官方 `batch-ara-backup-automation` 当前已支持：
  - 批量复制当前选中轨道
  - 将复制出的备份轨重命名为 `.bak`
  - 将这些备份轨隐藏并 inactive
- 当前这版只落版备份链，未继续把 ARA render 段写进正式发布说明

### Track 操作内核封装补齐

- 新增 track hidden / inactive 状态操作
- 新增批量 track state 更新封装
- 官方批量备份 automation 插件直接复用这一层能力，而不是把状态操作散落在插件里重复实现

## 修复

- 修复 automation extension management 页面动作接线，确保安装、刷新、启停、卸载操作走到共享宿主回调
- 修复 `Workflow Extensions` / `Automation Extensions` 设置页在固定壳层里的滚动结构
- 修复 bundled Python runtime 在 Tauri 资源准备阶段的 staging 链，避免打包前资源目录缺失

## 升级说明

- 本次没有引入新的桌面宿主架构切换，桌面主线仍然是 `Tauri + Node sidecar + Python FastAPI`
- 当前正式目标 DAW 仍然是 `Pro Tools`
- 插件协议版本基线已经更新到 `0.3.1`
