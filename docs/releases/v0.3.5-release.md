# Presto 0.3.5

`0.3.5` 是当前正式发布版本。这一版不改插件公开协议形状，重点是把宿主和后端的架构边界进一步收口，并修正两个已经会影响运行链正确性的状态一致性问题。

## Highlights

- 后端新增 `backend/presto/application/daw_runtime.py`，把 `target_daw -> runtime dependencies` 的解析独立成显式接缝；当前仍只接通 `pro_tools`，但多 DAW 扩展不再需要把新依赖散落到 container 装配入口各处。
- `build_service_container()` 现在统一通过 runtime resolver 解析 `daw`、`mac_automation` 与 `daw_ui_profile`，而不是继续在 service container 里硬编码 Pro Tools 运行时对象。
- DAW target 列表现在进入 `packages/contracts-manifest/daw-targets.json`，并通过 `scripts/generate-contracts.mjs` 生成 TypeScript / Python / Rust 三端共享产物，避免 contracts、backend 和 Rust runtime 再各自维护一份手写常量。
- capability 执行路径简化为“capability definition -> direct handler registry -> single execution context”，后端 handler 分发更直接，边界更清晰。
- `HostShellApp` 已经收口为 React 宿主组合层；偏好、导航和桌面插件目录装配分别下沉到独立 hook，宿主界面和运行时编排边界更清楚。
- `backend.daw-target.set` 现在会原子执行 `stop -> set target -> start -> wait ready`，切换目标 DAW 后不会把后续配置持久化链路切断。
- `system.health` 与 `daw.connection.getStatus` 回到纯查询语义，不再为了读状态隐式触发 DAW 连接建立。
- 插件目录刷新失败时，宿主会清空旧的 home / automation / pages 状态，并替换成干净的 error model，不再继续渲染陈旧插件。
- 历史路径 `frontend/electron/`、`frontend/sidecar/`、`frontend/runtime/` 已从当前源码主干移除；自动化资源源目录统一到 `frontend/tauri/resources/automation/`。
- Rust 桌面 runtime 已按领域拆成 `runtime.rs` 根调度器加 `runtime/backend.rs`、`runtime/plugins.rs`、`runtime/mobile_progress.rs` 三个模块，不再继续把所有宿主逻辑堆进单文件。

## Upgrade Notes

- 当前真实支持的 DAW 目标仍然只有 `pro_tools`；`logic`、`cubase`、`nuendo` 仍然只是类型预留。
- 这次版本把“DAW target 预留”和“当前真正支持”分开写进正式事实源：`reserved` 不等于 `supported`。
- 这次版本的公开边界没有扩成“多 DAW 已支持”，只是把 runtime dependency resolver 接缝做实。
- 宿主与插件公开协议面保持不变；这一版主要是内部架构整理和运行时一致性修复。
- 仓库根 `package.json` 现在是唯一手工维护的版本源；前端 `PRESTO_VERSION` 和后端 `VERSION` 都由同步脚本派生。
