# Presto 0.3.2

Presto `0.3.2` 是在 `0.3.1` 基础上，围绕多 DAW 复用前提补齐 capability canonical metadata、PTSL 扩展接入和版本面统一的一次内核更新。

## 亮点

### Public capability 层补齐 canonical DAW metadata

- capability manifest 与生成产物现在统一暴露 `supportedDaws`
- 每个公共 capability 现在声明 `canonicalSource`
- 每个公共 capability 现在声明 `fieldSupport`
- 后端 invoke 链路会按当前 DAW 对声明字段做显式校验，不再默默接受未声明字段

### Pro Tools 内部 PTSL catalog 与 runner 成形

- 新增内部全量 PTSL catalog 生成与加载链路
- Pro Tools adapter 通过 runner 做命令查找、版本 gate、dispatch 与错误归一
- 语义级 request shaping 继续留在 adapter，而不是把 runner 做成第二个超大 adapter

### Track toggle canonical baseline 收口

- `track.mute.set` / `track.solo.set` 改为 batch 语义
- 新增 `track.recordEnable.set`
- 新增 `track.recordSafe.set`
- 新增 `track.inputMonitor.set`
- 新增 `track.online.set`
- 新增 `track.frozen.set`
- 新增 `track.open.set`

### Developer Console 改为以后端 metadata 为事实源

- Console 不再继续把硬编码 inventory 当 capability 事实源
- runtime backend 新增 `listCapabilities()`
- Console 当前展示后端返回的 supported DAWs、canonical source 和 field support
- inventory 现在只保留 payload 默认值、状态和说明等 UI overlay 信息

### Runtime 日志与 Tauri 打包链收口

- 运行时日志改为按启动会话分文件写入，不再持续追加到单一 `current.log`
- sidecar RPC、bootstrap 和 backend supervisor 失败路径现在直接记录真实错误摘要
- Tauri DMG 打包链改为走 `hdiutil makehybrid + hdiutil convert -format UDZO`
- 当前 macOS `26.0.1` 环境下，`hdiutil create` 会阻塞，因此不再作为 `0.3.2` 发布链路事实

## 升级说明

- 当前正式目标 DAW 仍然是 `Pro Tools`
- 多 DAW 复用仍以公共 capability 形状为边界，adapter 负责各自 DAW 转译
- App、workspace package、Tauri 与 FastAPI 版本基线统一到 `0.3.2`
