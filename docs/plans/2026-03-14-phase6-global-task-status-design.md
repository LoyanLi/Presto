# Presto Phase 6 进度与时间匹配优化设计（仅优化现有能力）

## Context
- 当前 Import 与 Export 已有页面内进度显示，但“进度百分比”和“真实耗时体感”存在偏差：
  - Import 总进度基于 `current_index/total`，不同阶段耗时差异大时会出现“前快后慢”或“卡进度”。
  - Export 进度按快照序号推进，单个快照耗时波动大时，百分比与剩余时间不匹配。
- 你已明确：**不做新功能，不新增页面/入口/全局组件**，只在现有基础上优化准确性。

## 用户约束
1. 只优化现有 Import/Export 进度与时间匹配。
2. 不新增全局任务中心、任务条、历史页等新能力。
3. 尽量小改动、低风险上线。

## 目标与非目标
### 目标
- 让“总进度百分比”更贴近真实剩余时间。
- 在当前页面内给出稳定的 ETA（预计剩余时间）。
- 进度展示连续、单调、无明显回跳。

### 非目标
- 不改变 Import/Export 业务执行顺序。
- 不引入后台持久化统计系统。
- 不新增任何新导航与新页面。

## 优化方案（在现有代码上增量）

### 1) Import：从“纯数量进度”升级为“阶段加权进度”
现状：
- 后端提供 `stage`、`stage_progress` 与总 `progress`，但总进度仍偏数量视角。

优化：
- 保留现有字段与接口，新增/修正总进度计算逻辑：
  - `overall_progress = completed_stage_weight + stage_progress * current_stage_weight`
- 默认权重（可配置，先写死后再抽配置）：
  - `import_rename`: `0.55`
  - `color_batch`: `0.10`
  - `strip_silence`: `0.35`
- 约束：
  - 进度单调递增（不回退）。
  - 终态强制归 100%。

预期收益：
- 用户看到的总进度与真实时间分布更一致，尤其在 strip 阶段不再“后半程突然变慢”。

### 2) Export：从“快照序号进度”升级为“阶段内推进 + 平滑”
现状：
- 导出进度主要按快照数量推进，单快照耗时波动导致体感不准。

优化：
- 仍使用现有 `/export/status/{task_id}`，不改页面结构。
- 后端在每个快照内部关键步骤更新细粒度进度（例如：应用快照、等待稳定、执行导出、收尾）。
- 前端仅做轻度平滑显示（EMA 或限速推进），避免“长时间不动然后大跳”。

预期收益：
- 导出进度更连续，用户更容易判断任务是否正常运行。

### 3) ETA（预计剩余时间）：基于当前 run 的实时估计
算法（Import/Export 通用）：
- 当 `progress` 在 `(1, 99)` 时：
  - `elapsed = now - started_at`
  - `eta_raw = elapsed * (100 - progress) / progress`
- 对 ETA 做平滑与边界处理：
  - 限制最大值（避免异常夸大）。
  - 使用滑动平均，减少秒级抖动。
  - 终态清零。

展示方式：
- 仅在现有进度区域增加一行文案（如“预计剩余 xx:xx”）。
- 不新增组件类型，不改变页面信息架构。

## 数据与接口改动范围（最小化）
1. Import API
- 尽量复用现有 `progress/stage/stage_progress/started_at`。
- 若必要，仅新增可选字段 `eta_seconds`（向后兼容）。

2. Export API
- 复用现有 `progress/current_snapshot/total_snapshots/started_at`。
- 可选新增 `eta_seconds`（向后兼容），否则前端本地计算。

3. Frontend
- Import: `ImportWorkflow` 只调整总进度与 ETA 显示逻辑。
- Export: `ExportPanel` 只调整进度推进与 ETA 显示逻辑。

## 验收标准
1. Import 在三阶段执行中，总进度曲线与阶段耗时分布基本一致，无明显卡死假象。
2. Export 进度条在长任务中连续推进，跳变明显减少。
3. ETA 在运行中可读且稳定，误差随任务推进逐步收敛。
4. 全流程不新增页面/入口/全局组件。

## 风险与缓解
1. 风险：固定权重在不同工程类型下偏差。
- 缓解：先采用保守默认值，后续通过日志统计微调。

2. 风险：ETA 在低进度初期抖动大。
- 缓解：`progress < 5%` 时显示“计算中”，达到阈值后再展示。

3. 风险：前后端双侧平滑导致延迟感。
- 缓解：平滑只放前端，后端提供真实原始进度。

## 里程碑建议
- M1：Import 阶段加权进度落地 + 单测。
- M2：Export 细粒度进度点落地 + 单测。
- M3：Import/Export ETA 计算与展示落地。
- M4：真实工程回归（小/中/大项目）并调整权重参数。
